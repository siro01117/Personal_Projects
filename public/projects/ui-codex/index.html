normal 1.6

긴 본문에는 넉넉한 행간이 필요하다. 줄이 길수록 다음 줄을 찾기 쉽게 간격을 벌려야 한다.

loose 2.0

행간이 너무 넓으면 문단이 한 덩어리로 안 묶이고 줄이 흩어져 보인다.

큰 글자는 살짝 좁히고(-0.02em), 작은 대문자 라벨은 넓힌다(+0.05~0.1em). 본문은 건드리지 않는 게 안전하다.

강조 (Emphasis)본문 안에서 단어를 도드라지게

텍스트 강조는 굵게가 기본이다. 기울임은 한글에서 약하니 아껴 쓰고, 하이라이트는 정말 중요한 한 곳에만 둔다. 색 강조 포인트는 링크처럼 보이지 않게 주의한다. 취소된 값도 강조의 한 형태다.

한 문단에 강조가 셋 이상이면 어느 것도 강조가 아니다. 강조 수단은 한 번에 하나만 섞는다.

링크 (Link)기본·hover·방문·외부 링크 상태

실제 배포 전 한 컴포넌트만 반대 테마로 미리 본다. 토글은 이 카드 범위에만 적용되며, 두 테마에서 모두 읽히는지 확인하는 용도다.

--bad오류 · 실패 · 파괴적 동작
--info안내 · 진행 · 중립 알림

색을 미관이 아니라 의미에 묶는다. 한 번 정한 매핑은 화면 전체에서 동일하게 쓴다. 빨강을 어떤 화면에선 강조, 다른 화면에선 오류로 섞어 쓰지 않는다.

버튼

사용자의 행동을 유발하는 가장 기본 요소. 위계(primary/secondary/ghost), 위험도(danger), 크기, 상태(로딩·비활성)를 일관되게 다룬다.

위계 (Hierarchy)한 화면의 주행동은 primary 하나만

primary는 화면당 1개 권장. secondary는 채움 없는 외곽선, ghost는 배경까지 비워 밀도를 낮춘다. link형은 텍스트 흐름 안의 약한 동작에.

위험 동작 (Danger)되돌릴 수 없는 행동 강조

삭제·해지처럼 복구 어려운 동작에만. 채움형 danger는 최후 확정 버튼, soft형은 위험하지만 한 단계 더 확인이 남은 경우에.

크기 (Size)sm · md · lg

md가 기본. sm은 테이블·툴바 같은 조밀한 영역, lg는 모바일 주행동이나 랜딩 CTA에. 한 그룹 안에서는 크기를 섞지 않는다.

아이콘 + 텍스트의미를 보조하는 단색 아이콘

아이콘은 텍스트 의미를 강화하는 보조 수단. 방향성 아이콘(다음·이전)은 글자 뒤에 둔다. 아이콘만으로 동작을 추측하게 하지 않는다.

아이콘 버튼 (Icon only)공간 절약 · 라벨은 툴팁으로

라벨을 텍스트로 못 다는 만큼 반드시 data-tip(또는 aria-label)을 붙인다. 익숙한 동작(편집·삭제·공유)에만 쓰고 모호한 아이콘은 피한다.

상태 (States)hover · active는 실제로 만져보기

호버하면 밝아지고 누르면 살짝 가라앉는다. disabled는 커서·투명도로 명확히 구분하되, 왜 막혔는지는 별도 안내가 필요할 수 있다.

로딩 (Loading)클릭 → 처리 중 잠금

제출 직후 중복 클릭을 막는다. 로딩 중에는 비활성+스피너로 바꾸고 너비는 유지해 레이아웃이 튀지 않게 한다. (클릭해 확인)

버튼 그룹 (Group)붙은 동작들 · 관련 액션 묶음

서로 연관된 동작을 시각적으로 붙여 한 묶음임을 보인다. 모서리는 양 끝만 둥글린다. 단일 선택이 필요하면 세그먼트를 쓴다.

세그먼트 (Segmented)하나만 선택되는 탭형 토글

상호배타 선택(라디오 성격)에 쓴다. 항목 2~5개가 적당. 선택값은 항상 하나 유지되며 빈 상태가 없어야 한다. (클릭해 전환)

토글 (Toggle)눌린 상태가 유지되는 버튼

켜짐/꺼짐이 유지되는 동작에. aria-pressed로 상태를 노출하고 켜진 상태는 색·배경으로 또렷이 보인다. (클릭해 토글)

전체 너비 (Full width)모바일 · 폼 제출에 적합

좁은 폭에서 주행동을 키워 누르기 쉽게 한다. 한 영역에 full-width를 여러 개 쌓으면 위계가 흐려지니 위계 차이를 유지한다.

분할 버튼 (Split)기본 동작 + 추가 선택

가장 흔한 동작은 왼쪽 본체에서 바로, 변형은 오른쪽 화살표 메뉴에서. 본체와 화살표의 동작이 헷갈리지 않게 구분 선을 둔다. (화살표 클릭)

입력 · 폼

사용자가 값을 직접 넣는 모든 컨트롤. 라벨 · 도움말 · 에러 상태를 한 묶음으로 다루고, 포커스링으로 현재 위치를 항상 드러낸다.

텍스트 입력한 줄 자유 입력 · 라벨 + 도움말
실명으로 입력한다.

가장 기본. 라벨은 항상 입력 위에 붙이고 placeholder를 라벨 대용으로 쓰지 않는다. 포커스 시 테두리와 포커스링이 동시에 켜진다.

비밀번호표시·숨김 토글 포함
표시 아이콘으로 입력값을 잠깐 확인할 수 있다.

접미 아이콘 버튼은 글 라벨이 없으니 data-tip과 aria-label로 의미를 준다. 토글하면 type이 password ↔ text로 바뀐다.

검색아이콘 접두 + 지우기
입력이 있으면 지우기 버튼이 나타난다.

검색은 type="search"로. 접두 아이콘은 장식, 지우기 버튼은 동작이라 data-tip을 단다. 빈 값이면 지우기 버튼을 숨긴다.
